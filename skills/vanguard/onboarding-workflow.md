# Vanguard — Onboarding Workflow

## Objetivo

Guiar novo cliente através do onboarding de forma fluida, concisa e com alto empathy.

## Princípios

1. **Fluido** — Sem fricção, o cliente não percebe que é um "processo"
2. **Conciso** — Respeito pelo tempo do cliente
3. **Empathy** — Entender contexto antes de agir
4. **Feedback Loop** — Cliente sabe que sua opinião importa

## Steps

### Step 1: Primeiro Contato

```
Cliente chega via website/form/indicacao
    │
    ▼
Vanguard cumprimenta
    │
    ├── "Olá [Nome], sou o Vanguard da izzi Spark"
    ├── Entender contexto básico (nome, empresa, necessidade)
    └── Definir próximo passo junto
```

**Duração ideal:** 5 minutos max

### Step 2: Context Gathering

```markdown
Coletar info via conversa natural:

• O que cliente faz?
• Qual problema está tentando resolver?
• Já usa ferramentas similares?
• Qual o tamanho da operação?
• Qual o budget?

Não formulário - conversa.
```

### Step 3: Introduce Ecosystem

```
Apresentar izzi Spark de forma natural:
- O que fazemos
- Como podemos ajudar
- O que esperar do processo

Adaptar linguagem ao cliente (técnico vs não-técnico)
```

### Step 4: Setup Inicial

```
Se cliente quer proceder:
├── Configurar acesso
├── Definir ponto de contato
├── Estabelecer expectativas (timeline, output)
└── Documentar contexto no sistema
```

### Step 5: Follow-up

```
24h após onboarding:
- Checar se cliente tem dúvidas
- Confirmar próximo passo
- Solicitar feedback sobre processo
```

## 6 Hats no Onboarding

| Hat | Aplicação |
|-----|-----------|
| White | Qual é a situação real do cliente? |
| Red | Como o cliente está se sentindo? |
| Black | O que pode dar errado? |
| Yellow | Como isso beneficia o cliente? |
| Green | Que alternativa podemos oferecer? |
| Blue | Qual é o próximo passo? |

## Anti-patterns

❌ Não enviar questionnaire gigante  
❌ Não fazer perguntas sem contexto  
❌ Não pressionar para venda  
❌ Não prometer o que não pode entregar  
❌ Não ignorar feedback inicial  

## Checkpoints

- [ ] Cliente entendeu o que é izzi Spark
- [ ] Cliente sabe o que esperar
- [ ] Próximo passo está claro para ambos
- [ ] Contexto documentado
- [ ] Feedback coletado
