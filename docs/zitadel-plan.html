Local Development Setup

Start Zitadel locally with Docker Compose to test before production.

# Start Zitadel + PostgreSQL locally
cd izzi-spark/zitadel
docker-compose up -d

# Watch logs
docker-compose logs -f zitadel

# Access UI at http://localhost:8080
⚠️ Prerequisite: Install Docker on your machine.
  • Zitadel runs on port 8080
  • PostgreSQL auto-creates and migrates schema
  • First login creates admin user
2

Configure Zitadel

Create organization, configure Google OAuth provider, and setup roles.

2.1 Get Admin API Key

# Get from Zitadel UI or CLI
ZITADEL_KEY=$(docker exec zitadel-1 ./zitadel createdomain admin --raw)

2.2 Create izzi Spark Organization

# Via REST API
curl -X POST https://localhost:8080/admin/v1/orgs \
  -H "Authorization: Bearer $ZITADEL_KEY" \
  -d '{"name": "izzi-spark", "preferredDomain": "izzispark"}'

2.3 Configure Google OAuth (UI)

  • Go to: Settings → Identity Providers → Add → Google
  • Enter Client ID and Client Secret
  • Redirect URI: http://auth.izzispark.cloud:8080/idps/zitadel-generator/callback
3

Hermes Integration

Update auth service to proxy requests to Zitadel.

# hermes/.env
ZITADEL_URL=http://auth.izzispark.cloud:8080
ZITADEL_SERVICE_KEY=your_service_key
✅ Key Benefit: No more custom JWT handling. Zitadel provides standardized OAuth2 + OIDC.
4

Production Deploy (Dokploy)

Deploy to Hostinger KVM4 using Dokploy.

4.1 DNS Configuration

# Hostinger DNS records
auth.izzispark.cloud  A  YOUR_SERVER_IP

4.2 Deploy Steps

  1. Login to https://host.izzispark.cloud
  2. Create project "zitadel-auth"
  3. Upload docker-compose.yml
  4. Configure environment variables
  5. Dokploy auto-configures SSL
5

Google OAuth + Email

5.1 Google Cloud Console

  • Go to: https://console.cloud.google.com/apis/credentials
  • Create OAuth 2.0 Client ID
  • Redirect URI: http://auth.izzispark.cloud:8080/login/authucceeded

5.2 Resend Email Setup

ZITADEL_NOTIFICATIONS_SMTP_HOST smtp.resend.com
ZITADEL_NOTIFICATIONS_SMTP_PASSWORD re_your_api_key (from resend.com)

🔧 Troubleshooting

Zitadel won't start

# Reset database
docker-compose down -v
docker-compose up -d

Database connection failed

# Check PostgreSQL health
docker exec -it postgres_data pg_isready -U postgres

Google OAuth not working

Verify redirect URI matches exactly in Google Cloud Console.
